# 🏠 Квартирный бот для Telegram

Бот живёт в вашем с мужем чате. Кидаете ссылку на квартиру с realestate.com.au
или domain.com.au — он заходит через настоящий Chrome, собирает характеристики
(адрес, цена, спальни/санузлы/паркинг, время инспекций, агент, особенности) и
заносит в базу (Google Sheets). С ним можно говорить обычными словами:
«покажи всё, что ещё не смотрели», «отметь #3 как просмотрена, понравилась 9/10»,
«удали #5».

LLM под капотом — Claude. Отвечает по-русски.

---

## Что внутри

| Файл | Что делает |
|------|------------|
| `bot.py` | Точка входа, ловит сообщения в Telegram |
| `scraper.py` | Открывает страницу настоящим Chrome (patchright), собирает сырые данные |
| `llm.py` | Claude: вытаскивает характеристики из страницы + разговор |
| `sheets.py` | Чтение/запись/удаление в Google Sheets |
| `config.py` | Настройки из `.env` |

База — Google Sheets: вы оба можете открыть таблицу с телефона и править руками,
бот пишет в те же строки.

## Установка

### Шаг 1. Бот в Telegram
1. [@BotFather](https://t.me/BotFather) → `/newbot`. Получишь **токен** `123456:ABC...`.
2. **Важно для группы:** `/setprivacy` → выбери бота → **Disable** (иначе он не видит сообщения).
3. Добавь бота в ваш чат.

### Шаг 2. Ключ Claude
[console.anthropic.com](https://console.anthropic.com) → API Keys → `sk-ant-...`.

### Шаг 3. База Google Sheets
1. Создай пустую таблицу на [sheets.google.com](https://sheets.google.com).
   Из адреса `docs.google.com/spreadsheets/d/`**`ЭТО_ID`**`/edit` скопируй ID.
2. Сервисный аккаунт (бесплатно):
   - [console.cloud.google.com](https://console.cloud.google.com) → создай проект.
   - Включи **Google Sheets API**.
   - Credentials → **Create credentials → Service account**.
   - У аккаунта: **Keys → Add key → JSON**. Файл переименуй в `credentials.json`, положи рядом с `bot.py`.
3. Открой `credentials.json`, скопируй email `...@...iam.gserviceaccount.com`
   и **поделись таблицей** с ним (Share → Editor).

### Шаг 4. Настройки
```bash
cp .env.example .env
```
Впиши в `.env`: `TELEGRAM_TOKEN`, `ANTHROPIC_API_KEY`, `SHEET_ID`.

### Шаг 5. Зависимости и Chrome
```bash
cd realestate-bot
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
patchright install chromium   # на всякий случай; бот использует системный Chrome
```
Нужен установленный **Google Chrome** (бот запускает именно его).

### Шаг 6. Проверка
```bash
python bot.py
```
Кинь ссылку на квартиру — придёт карточка, в таблице появится строка. Ctrl+C чтобы остановить.

## Запуск 24/7 (автозапуск на Mac)

Парсинг realestate.com.au работает только **с домашнего IP** (см. ниже), поэтому
бот живёт локально, а не в облаке. Чтобы он поднимался сам и пережил перезагрузки —
через `launchd`.

Файл автозапуска — `deploy/com.tanya.realestate-bot.plist` (копия лежит в
`~/Library/LaunchAgents/`). Он запускает бота под `caffeinate -i -s`, поэтому
**пока бот работает, Mac не засыпает**. `KeepAlive` поднимает бота при падении и
сетевых сбоях, `RunAtLoad` — при логине. Команды:

```bash
# установить (один раз) и запустить
cp deploy/com.tanya.realestate-bot.plist ~/Library/LaunchAgents/
launchctl load -w ~/Library/LaunchAgents/com.tanya.realestate-bot.plist

# остановить
launchctl unload ~/Library/LaunchAgents/com.tanya.realestate-bot.plist

# живые логи
tail -f ~/realestate-bot/bot.log
```

Условия «всегда работает»:
- Mac **включён**, ты **залогинена** (Chrome нужен рабочий стол).
- Желательно **в розетке** — `caffeinate -s` держит от сна на питании от сети.
- Если закрыть крышку — ноут всё равно уснёт (clamshell). Для 24/7 держи крышку
  открытой или подключи внешний монитор. Выключен — бот не работает; включишь —
  поднимется сам.

## Как пользоваться

**Добавить квартиру** — кинь ссылку в чат. Можно **сразу много ссылок одним
сообщением** с заметками между ними:
```
G10/15 Ralph Street, Alexandria  https://realestate.com.au/...
https://realestate.com.au/...
я сделал реквест на просмотр
```
Бот обработает каждую ссылку, адрес с той же строки подложит в данные, а строку-
заметку прицепит к предыдущей квартире. Дубли узнаёт по ссылке.

**Спросить / изменить / удалить** — упомяни бота (`@имя_бота ...`), ответь на его
сообщение или пиши в личку:
- «что мы ещё не ходили смотреть?»
- «какие до $800 в неделю в Bondi?»
- «отметь #4 как viewed, дата сегодня, оценка 8/10»
- «#2 — reject, далеко от работы»
- «добавь к #5 заметку: солнечная, но шумная улица»
- «#1 — 2 спальни, 1 санузел, $750 в неделю»
- «удали #3»

Статусы: `new`, `interested`, `shortlisted`, `viewed`, `rejected`, `applied`.
«Отклонить» меняет статус на `rejected` (квартира остаётся), «удалить» — стирает строку.

**Перепроверить квартиру** — «проверь, не изменилась ли цена у #15» — бот заново
откроет страницу и обновит данные.

**Авто-проверка раз в день** — каждый день в 09:00 (Сидней) бот сам обходит все
активные квартиры (кроме `rejected`), обновляет цены/инспекции и пишет в чат, что
изменилось. Если изменений нет — молчит. Время задаётся в `bot.py` (`DAILY_CHECK_TIME`).

## Почему локально, а не в облаке

**realestate.com.au сидит за Akamai Bot Manager** и режет автоматический доступ
(HTTP 429). Проверено: не проходят ни curl, ни Jina, ни headless-браузер.

Решение — настоящий Google Chrome через **patchright** (стелс-версия Playwright):
он проходит JS-проверку Akamai и отдаёт полную страницу. Окно открывается за
пределами экрана, профиль с куками сохраняется в `.chrome-profile/` (первый запрос
после перезагрузки чуть медленнее, дальше быстро).

Ключевое: Akamai пускает «человеческие» IP и блокирует дата-центры. Поэтому это
работает **только с домашнего IP**. В облаке (Railway/Render/VPS) realestate снова
заблокирует — там понадобился бы платный скрапер с резидентными прокси
(`ZENROWS_API_KEY` в `.env` включает такой fallback).

domain.com.au защищён слабее.
